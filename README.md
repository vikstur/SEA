# SEA


Система за управление на фитнес (Fitness Management System)
Това е цялостно Windows Forms приложение, създадено за управление на фитнес зали, абонаменти на клиенти и ежедневно присъствие. Системата използва Entity Framework Core за управление на базата данни и BCrypt.NET за сигурна автентикация на потребителите.

## Функционалности
👤 Управление на потребители
Сигурна автентикация: Хеширане на пароли чрез BCrypt за максимална защита.

Достъп според ролята: Поддръжка на роли за Клиенти и Персонал/Администратори.

Търсене на клиенти: Възможност за търсене и филтриране на клиенти по име или имейл.

💳 Система за абонаменти
Различни планове: Създаване и управление на различни видове абонаменти (напр. Фитнес, СПА, Басейн).

Заявки за покупка: Клиентите могат да заявяват нови абонаменти, които персоналът може да одобри или отхвърли.

Автоматично изтичане: Системата проследява крайните дати и броя посещения, като автоматично маркира абонамента като "Изтекъл" при достигане на лимита.

Услуги по нива: Свързване на специфични услуги с определени типове планове.

📅 Присъствие и проследяване
Система за чекиране: Проследяване на влизанията на клиентите в реално време.

Лимит на посещенията: Автоматично намаляване на оставащите посещения за "Базови" планове или планове с ограничен брой влизания.

История на посещенията: Клиентите могат да виждат своята лична история, а персоналът – общия трафик в залата.

📊 Справки и отчети
Дневни дневници: Преглед на посещенията за конкретна дата.

Периодични отчети: Генериране на справки за посещаемостта за избран период от време.

## Технологичен стек
Рамка (Framework): .NET / Windows Forms

ORM: Entity Framework Core (Database First)

База данни: Microsoft SQL Server

Сигурност: BCrypt.Net-Next

Инструменти: EF Core Power Tools (за генериране на моделите)

## Структура на базата данни
Системата разчита на релационна база данни FitnessManagementDB, състояща се от следните основни таблици:

Users: Съхранява потребителски данни и профили.

Subscriptions: Проследява активните планове на клиентите.

SubscriptionTypes: Дефинира наличните планове (цена, продължителност, лимит на посещения).

Services: Каталог с предлаганите услуги (Фитнес, СПА и др.).

Attendances: Логове за всяко влизане на потребител в обекта.

PurchaseRequests: Опашка за чакащи одобрение заявки за абонамент.

## Инсталация и настройка
Конфигурация на базата данни:

Уверете се, че SQL Server е стартиран.

Стрингът за връзка (connection string) по подразбиране сочи към DESKTOP-CBNDVHL\SQLEXPRESS. Актуализирайте метода OnConfiguring в FitnessManagementDBContext.cs, за да съответства на вашия локален сървър.

Зависимости (NuGet пакети):
Инсталирайте следните пакети:

Microsoft.EntityFrameworkCore.SqlServer

BCrypt.Net-Next

Миграции:
Ако започвате с празна база данни, използвайте Package Manager Console:

PowerShell
Update-Database
## Структура на проекта
/Models: Съдържа EF Core ентитата и DbContext класа.

/Services: Съдържа бизнес логиката на приложението:

UserServices: Регистрация, вход и търсене на клиенти.

SubscriptionServices: Одобряване на заявки и управление на планове.

AttendanceServices: Управление на чекирането и брояча на посещения.

VisitsServices: Справки и проследяване на историята.

/Core: Споделено управление на сесиите и помощни класове.
